# DIO_NotebookLM
Repositório do Desafio DIO - NotebookLM

## Sobre o projeto

Neste projeto, utilizei o NotebookLM como ferramenta de apoio para estudar **Market Basket Analysis (Análise de Cesta de Compras)**.
A proposta foi ir além de apenas ler conteúdos. Busquei aprender de forma prática, organizando informações, testando perguntas (prompts) e consolidando tudo em um material simples e reutilizável.

## Objetivo

O objetivo principal foi entender como funciona a análise de cesta de compras e como ela pode ser aplicada no varejo.
Durante o processo, também foquei em:
- Melhorar a forma de fazer perguntas para IA  
- Validar informações com diferentes fontes  
- Organizar o aprendizado de forma clara  

## Fontes utilizadas

Para construir o material, utilizei algumas fontes confiáveis:
- Artigos sobre Market Basket Analysis  
- Documentação da biblioteca `mlxtend` (Python)  
- Conteúdos técnicos de Data Science  
- Materiais sobre o algoritmo Apriori  

## Uso da IA e testes de prompts

Uma parte importante do projeto foi testar diferentes formas de fazer perguntas.
Alguns exemplos:
"Explique o algoritmo Apriori de forma simples
Como aplicar Market Basket Analysis no varejo?
Quais são as limitações dessa técnica?
Me dê exemplos com suporte, confiança e lift"

Com o tempo, percebi que:
- Perguntas genéricas geram respostas superficiais  
- Perguntas mais específicas trazem respostas melhores  
- Pedir exemplos ajuda muito no entendimento  

Também enfrentei algumas dificuldades:
- Algumas respostas não eram tão claras  
- Foi necessário validar informações  
- Ajustar os prompts fez bastante diferença  

O principal aprendizado foi simples:  
**quanto melhor a pergunta, melhor a resposta.**

## Resumo do conteúdo

Market Basket Analysis é uma técnica que identifica padrões de compra, mostrando quais produtos costumam ser comprados juntos.
É muito utilizada em:
- Varejo  
- E-commerce  
- Estratégias de venda (cross-sell e upsell)  

Os principais conceitos são:
- **Suporte**: frequência de um conjunto de itens  
- **Confiança**: probabilidade de compra conjunta  
- **Lift**: força da relação entre os itens  

O principal algoritmo utilizado é o **Apriori**, que identifica combinações frequentes de produtos.

## Glossário

- **Itemset**: conjunto de itens  
- **Regra de associação**: relação entre produtos (ex: A → B)  
- **Suporte**: frequência no conjunto de dados  
- **Confiança**: probabilidade condicional  
- **Lift**: indica a força da associação  
- **Apriori**: algoritmo para gerar regras
